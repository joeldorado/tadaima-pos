## Hallazgos detallados

### Hallazgo #1 — Teléfono de cliente no es único
| Campo | Valor |
|-------|-------|
| **Severidad** | MEDIA |
| **QA** | QA-02 · B4 |
| **Módulo** | CustomerController / StoreCustomerRequest |
| **Síntoma** | `POST /customers` con teléfono duplicado devuelve `201 Created` en lugar de `422` |
| **Impacto** | Posibles clientes duplicados con mismo número de teléfono |
| **Fix sugerido** | Agregar `'unique:customers,phone'` en `StoreCustomerRequest::rules()` para el campo `phone` |

### Hallazgo #2 — `reserved_count` no disminuye al cancelar folio
| Campo | Valor |
|-------|-------|
| **Severidad** | BAJA / A DEFINIR |
| **QA** | QA-03 · A4 |
| **Módulo** | PreSaleOrderService::cancel() |
| **Síntoma** | Al cancelar un folio, `reserved_count` en el catálogo sigue igual (no baja) |
| **Nota** | Puede ser intencional (el slot se consume históricamente aunque se cancele) o un bug. **Requiere decisión de negocio**. |
| **Fix si es bug** | Decrementar el conteo de unidades activas al cancelar en `PreSaleOrderService::cancel()` |

### Hallazgo #3 — Anticipo mayor al precio total es aceptado
| Campo | Valor |
|-------|-------|
| **Severidad** | ALTA |
| **QA** | QA-03 · B6 |
| **Módulo** | PreSaleOrderService::createOrder() / StorePreSaleOrderRequest |
| **Síntoma** | Folio AirPods ($6,499 total) con `advance_amount: $10,000` devuelve `200 OK` — el anticipo excede el precio |
| **Impacto** | Datos financieros inconsistentes; balance negativo posible |
| **Fix sugerido** | Agregar validación en `PreSaleOrderService::createOrder()`: si `advance_amount > total_price`, lanzar `DomainException` |

### Hallazgo #4 — Sin control de permisos en catálogos por rol
| Campo | Valor |
|-------|-------|
| **Severidad** | ALTA |
| **QA** | QA-03 · C1, C2 |
| **Módulo** | PreSaleCatalogsController / Middleware de permisos |
| **Síntoma** | El cajero puede `POST /pre-sale-catalogs` (crear catálogos) y `PATCH /pre-sale-catalogs/{id}/status` (publicar/cancelar) — solo admin debería poder |
| **Impacto** | Cajero puede crear productos de preventa no autorizados o cancelar catálogos activos |
| **Fix sugerido** | Agregar middleware `can:manage-pre-sale-catalogs` en las rutas, o verificar rol en el controller |

### Hallazgo #5 — Cajero puede ver folios de otras sucursales
| Campo | Valor |
|-------|-------|
| **Severidad** | MEDIA |
| **QA** | QA-03 · C3 |
| **Módulo** | PreSaleOrdersController::index() |
| **Síntoma** | Cajero Centro consulta `?store_id=2` (Macroplaza) y obtiene 2 folios. Debería ver solo su tienda. |
| **Impacto** | Exposición de datos de clientes entre sucursales |
| **Fix sugerido** | En `PreSaleOrdersController::index()`, si el usuario tiene rol `cajero`, forzar `store_id` a `$request->user()->store_id` |
