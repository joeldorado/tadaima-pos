## Lo que TÚ (Joel) necesitas correr antes/durante el rollout

### 1. SQL: limpiar drafts huérfanos en producción (Bug 14)

Antes de subir la nueva versión, limpia drafts viejos que están reservando inventario y comiendo el cupo de cajeros:

```sql
-- Ver cuántos drafts hay zombies (sin commit)
SELECT
  status,
  COUNT(*) AS total,
  MIN(created_at) AS más_viejo,
  MAX(created_at) AS más_nuevo
FROM sales_drafts
WHERE status IN ('open', 'suspended')
GROUP BY status;

-- Cancela drafts sin items con >30 min de creados
UPDATE sales_drafts
SET status = 'cancelled', updated_at = NOW()
WHERE status IN ('open', 'suspended')
  AND created_at < NOW() - INTERVAL '30 minutes'
  AND id NOT IN (SELECT DISTINCT draft_id FROM sales_draft_items);

-- Cancela drafts con items pero sin actividad >6 horas
UPDATE sales_drafts
SET status = 'cancelled', updated_at = NOW()
WHERE status IN ('open', 'suspended')
  AND updated_at < NOW() - INTERVAL '6 hours'
  AND id IN (SELECT DISTINCT draft_id FROM sales_draft_items);
```

> Si usas MySQL en lugar de Postgres, cambia `INTERVAL '30 minutes'` por `INTERVAL 30 MINUTE`.

> Después del deploy, el comando `drafts:cleanup` corre automáticamente cada hora vía Laravel Schedule (configurado en `routes/console.php`). Verifica que el worker del scheduler esté corriendo en Cloud Run.

### 2. Comando manual de limpieza (cuando quieras)

```bash
# Ver qué cancelaría sin hacerlo
php artisan drafts:cleanup --dry-run

# Cancelar
php artisan drafts:cleanup
```
